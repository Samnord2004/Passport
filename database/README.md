# База данных PostgreSQL: Модели SQLAlchemy и Миграции Alembic

В данном каталоге находятся модели базы данных и структуры миграций, полностью соответствующие текущей схеме данных веб-платформы **«Цифровой паспорт объекта»**.

## Структура каталога

* **`models.py`** — SQLAlchemy (v2.0+) декларативные модели для всех 8 таблиц системы:
  * `User` (`users`) — Пользователи платформы (Администраторы, Операторы, Собственники, Специалисты).
  * `BuildingObject` (`building_objects`) — Объекты недвижимости.
  * `ChecklistTemplate` (`checklist_templates`) — Шаблоны чек-листов ТО.
  * `Schedule` (`schedules`) — Календарный график регламентного технического обслуживания (ТО).
  * `CompletedChecklist` (`completed_checklists`) — Завершенные обслуживание отчеты и опросные чек-листы.
  * `NotificationLog` (`notification_logs`) — Логи уведомлений (Telegram, Email, VK, MAX).
  * `SystemSettings` (`system_settings`) — Настройки синхронизации и уведомлений.
  * `Session` (`session`) — Управление сессиями пользователей.
* **`alembic.ini`** — Конфигурационный файл утилиты миграций Alembic.
* **`alembic/env.py`** — Скрипт окружения Alembic, подключающий SQLAlchemy метаданные моделей и автоматически загружающий параметры соединения из переменных окружения (`DATABASE_URL`).
* **`alembic/versions/3a2b1c4d5e6f_initial_migration.py`** — Скрипт первой миграции Alembic для развертывания полной схемы в пустую БД PostgreSQL.

## Как использовать миграции Alembic

Для применения данных миграций на сервере с Python установите необходимые зависимости:

```bash
pip install sqlalchemy alembic psycopg2-binary
```

Запустите команду применения миграций вверх:

```bash
# Alembic автоматически извлечет строку подключения из переменной окружения DATABASE_URL
export DATABASE_URL="postgresql://postgres:password@localhost:5432/tech_passport"

alembic upgrade head
```

## Интеграция с Живым Сервером (TypeScript + Node.js)

Основной сервер нашего приложения (`server.ts` и `server/data-store.ts`) и сессионный модуль разработаны на TypeScript в целях максимального быстродействия веб-карты и интерактивного UI. 

Для полной готовности к запуску в контейнере или на сервере, в TypeScript-backend **встроена автоматическая система применения миграций и инициализации PostgreSQL**. При наличии переменной `DATABASE_URL` или переменных конфигурации `DB_HOST`, `DB_PORT`, платформа:
1. Автоматически создает пул подключений (`pg.Pool`).
2. Принудительно разворачивает все таблицы в базе данных PostgreSQL (если они отсутствуют).
3. При необходимости наполняет базу данных демонстрационными данными (сидирование).
4. Настраивает защищенное хранение сессий Express в таблице `session` с автоочисткой просроченных авторизаций.

Вы можете переключить базу данных на PostgreSQL, сохранив все данные в надежном хранилище, передав параметры авторизации через переменные окружения, указанные в файле `.env`.
